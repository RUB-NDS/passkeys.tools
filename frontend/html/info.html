Welcome to the Passkey-Workbench

Passkey-Workbench is a comprehensive development and debugging tool for WebAuthn/Passkey implementations. It provides a suite of utilities to help developers understand, test, and verify the security of passkey authentication flows.

Key Features:
  • Create & Get Operations: Generate WebAuthn credential creation and assertion requests for testing
  • Attestation & Assertion Analysis: Decode and inspect attestation objects and assertion responses
  • Key Management: Generate, store, and manage cryptographic keys for testing different scenarios
  • User Management: Create and manage test users for different relying parties
  • Interceptor: Intercept and modify live passkey operations with preconfigured security tests including context manipulation, challenge reuse, and origin validation
  • History: Track all intercepted requests and responses
  • Converters: Convert between different encoding formats (Base64, Base64URL, Hex)
Getting Started:
  1. Use the Create tab to generate credential creation requests
  2. Use the Get tab to generate credential get requests
  3. Analyze responses in the Attestation and Assertion tabs
  4. Manage test credentials in the Keys and Users tabs
  5. Use the Interceptor with the browser extension to debug real passkey flows

Current Limitations
  • Only the None and Packed attestation statement formats are supported.
  • In the Packed attestation statement format, the signature and certificates need to be added manually if required.
  • Extensions are not automatically parsed but need to be added manually if required.
  • Client Extension Processing and Authenticator Extension Processing is not supported.
  • To use the Ed25519 algorithm in Chrome, you need to enable the #enable-experimental-web-platform-features flag.
